# 面试口述稿（模板）

> 目标：1 分钟内讲清楚 “它做什么 + 主流程 + 设计原因”

## 一句话总结（填空）

“这个模块/函数用于在 GPU 上计算稳定关键点并返回，核心目标是从图像中筛选出可用于后续跟踪/插值的可靠特征点。”

## 主流程（5~8 步）

1. 检查输入图像是否为空
2. 计算窗口/矩阵尺寸并做容量校验
3. CPU 端转灰度 8U
4. H -> D 拷贝到 GPU
5. Kernel1：8U 转 float 图像
6. Kernel2：窗口内计算关键点与质量分数
7. D -> H 回传 + CPU 端组装 KeyPoint
8. 返回结果（成功/失败）

## 关键设计原因（2~4 点）

- 为什么用 GPU：窗口级计算量大，GPU 并行效率更高
- 为什么要分两次扫描：先算对比度/缓存数据，再算质心与梯度，减少全局访问并提升稳定性
- 为什么需要质量过滤：剔除低对比/边缘型点，保留更稳定的关键点
- 为什么做亚像素细化：提高关键点定位精度，提升后续匹配/跟踪稳定性

## 结果与价值（1 句话）

“最终得到稳定的关键点集合，用于后续的匹配、跟踪或光流估计。”
